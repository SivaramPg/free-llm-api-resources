# Design: freellm.sivaram.dev — Astro UI for free-llm-api-resources

Date: 2026-07-21
Status: Approved (pending final spec review)

## Goal

A fast, visually distinctive static site presenting the free LLM API provider list
from this repo's generated `README.md`. Hosted on Cloudflare at **freellm.sivaram.dev**.
Purpose: best-in-class UX for this data + search traffic to sivaram.dev.

## Constraints & decisions (agreed)

- **Data source:** parse `README.md` at build time. Never modify the Python
  generator — upstream (cheahjs) sends automated README-update PRs and they must
  merge cleanly. Every merge to main redeploys the site with fresh data.
- **Structure:** hub page + one static page per provider (`/providers/[slug]`).
  ~27 indexable pages.
- **Brand colors:** extracted from favicons via saturation-weighted quantization
  (not naive average), with a manual override map. Light/dark tokens derived
  programmatically with contrast enforcement.
- **Screenshots:** provider homepages only (26), captured in **both light and dark
  themes** where the site supports it (fall back to single capture otherwise).
  Stored in `site/public/screenshots/`, served free by Cloudflare.
- **Search:** client-side fuzzy model search (MiniSearch) across all providers —
  the headline feature the README can't offer.
- **Attribution:** footer credit + link to github.com/cheahjs/free-llm-api-resources.
- **Canonical domain:** `https://freellm.sivaram.dev` (OG tags, sitemap, canonical URLs).
  Exact-match hostname acknowledged as a negligible ranking factor — ranking work
  happens at page level (see SEO section).

## Data layer

`parse-readme.ts` runs at Astro build time:

1. Read `../README.md`, split on `## Free Providers` / `## Providers with trial credits`,
   then `### [Name](url)` sections.
2. Per section, parse into a discriminated union by `layout` kind (declared in
   `provider-meta.ts`, keyed by slug):
   - **`per-model-table`** — HTML `<table>` of model → limits (Google AI Studio,
     Cerebras, Groq). Parse rows; split limits on `<br>` into structured
     `{ value, unit }` pairs (e.g. `250,000 tokens/minute`).
   - **`shared-quota`** — provider-level limits + markdown list of model links
     (OpenRouter, Cohere, Cloudflare Workers AI, GitHub Models).
   - **`credits`** — `**Credits:** $X` + models blurb/list (all trial providers).
   - **`minimal`** — prose notes + doc links (NVIDIA NIM, Vercel AI Gateway,
     HuggingFace, Mistral, OpenCode Zen).
3. Extract caveats into structured flags by pattern-match on the prose:
   `phoneVerification`, `dataTraining`, `geoRestriction`, plus free-text notes kept verbatim.
4. Validate the full result with zod. **Any parse failure or unknown provider
   fails the build loudly** — README drift surfaces at deploy time, never as a
   silently broken page. A new upstream provider requires adding one
   `provider-meta.ts` entry (build error names the missing slug).

Provider shape (abridged):

```ts
type Provider = {
  slug: string; name: string; url: string;
  category: 'free' | 'trial';
  layout: 'per-model-table' | 'shared-quota' | 'credits' | 'minimal';
  caveats: Caveat[];            // typed badges
  notes: string[];              // verbatim prose
  limits?: Limit[];             // provider-level (shared-quota)
  credits?: string;             // trial providers
  models: Model[];              // possibly empty (minimal)
};
type Model = { name: string; url?: string; limits?: Limit[] };
type Limit = { value: number | null; unit: string; raw: string };
```

## Asset pipeline (one-shot scripts, outputs committed)

- **`fetch-favicons.ts`** (tsx + cheerio): per provider homepage, collect
  `link[rel~=icon]`, `apple-touch-icon`, web-manifest icons; download all, keep the
  largest square image. Fallback chain: og:image → Google favicon API (256px) →
  generated letter-avatar SVG in brand color.
- **`extract-colors.ts`** (sharp): decode favicon → pixel quantization weighted by
  saturation × population, so vibrant brand hues win over white/gray backgrounds
  (Cloudflare → orange, Vercel → black). Manual `colorOverrides` map in
  `provider-meta.ts` beats extraction. Emits `brand.json`:
  per slug `{ base, light: { wash, border, text }, dark: { wash, border, text } }`,
  each derived variant contrast-checked (WCAG AA against the theme bg; text shades
  auto-lightened/darkened until passing).
- **Screenshots** (agent-browser, per runbook): 26 homepages at 1440×900, capture
  with `prefers-color-scheme` light AND dark; if the site doesn't respond to the
  scheme, keep the single capture for both slots. Output webp (quality ~80) to
  `public/screenshots/<slug>-{light,dark}.webp`. Site shows the variant matching
  the active theme; missing screenshot → brand-gradient placeholder.

Scripts are idempotent; re-run when a provider is added or rebrands.

## Pages & UI

**Hub `/`** — compact hero (title, one-liner, GitHub link), search box, then card
grid in two groups: Free Providers, Trial Credits.

Card: favicon + name, brand wash background (base color at low alpha composited
over theme bg), model count, headline limit (e.g. "50 req/day shared"), caveat
badges, subtle brand-colored border. Hover: card lifts and the homepage screenshot
fades in as a preview. Click → provider page.

**Provider `/providers/[slug]`** — screenshot hero (theme-matching variant) under a
brand-gradient overlay with favicon + name; caveat callouts; limits section; then
the bespoke model presentation:

- `per-model-table` → responsive table, limits as monospace pills per model
- `shared-quota` → shared-limit banner + model link grid
- `credits` → prominent credit amount + model list/blurb
- `minimal` → notes + prominent external doc links; renders a plain model list
  when models exist (e.g. OpenCode Zen)

CTA button ("Get API key →" to the provider console) in brand color. Prev/next
provider footer nav.

**Search** — MiniSearch index (model name + provider name + caveat flags) built at
compile time, shipped as JSON (~30–50 KB); instant fuzzy results grouped by model,
showing every provider carrying it with limits side by side. Filters: category,
no-phone-verification. Progressive enhancement: no JS → grid still browsable.

## Theme & typography

- Dark bg `#141416` family, light bg `#faf8f5` family; never pure black/white.
  Text: softened complementary shades (`#e8e6e3` on dark, `#26242f` on light).
- Default follows `prefers-color-scheme`; toggle persists to localStorage and sets
  `data-theme` on `<html>` (inline head script to avoid flash).
- All colors as CSS custom properties in `tokens.css`; per-provider brand vars
  injected as inline style from `brand.json`.
- Fonts self-hosted via Fontsource: **Mozilla Headline** (headings), **DM Sans**
  (body), **Geist Mono** (model IDs, limits). `font-display: swap`, preload woff2.

## Deploy

- Cloudflare Workers static assets (wrangler), project root `site/`,
  build `astro build`. Custom domain `llm.sivaram.dev`.
- Deploy on push to main via Cloudflare git integration → upstream README merges
  auto-refresh the site. Site build failure (parser drift) blocks deploy; last
  good version stays live.

## SEO

Hostname keyword is negligible for ranking; the plan is page-level:

- **Titles/descriptions:** per-provider pages target real queries — pattern
  `"{Provider} free tier: models, rate limits & how to get an API key"`;
  hub targets "free LLM API" head terms. Descriptions include the headline
  limit and model count (fresh data → fresh snippets each deploy).
- **Structured data (JSON-LD):** `ItemList` of providers on the hub;
  per-provider `WebPage` + `ItemList` of models; `BreadcrumbList` on provider
  pages.
- **Mechanical hygiene:** `@astrojs/sitemap`, canonical URLs on every page,
  OG + Twitter card tags (provider screenshots double as per-page OG images),
  robots.txt, semantic heading hierarchy, descriptive internal anchor text.
- **Freshness:** every upstream data merge redeploys with changed content and an
  updated sitemap `lastmod`.
- **Backlinks (post-launch):** PR to upstream README proposing a "browse as a
  website" link; share in relevant communities. Out of build scope but the
  highest-leverage ranking input.

**Site-own favicon + OG image:** designed and added **last**, after the UI's
visual identity settles (favicon, `og:image` for hub, `theme-color` meta).

## Error handling

- Parser: zod validation; fail build with named provider + line context.
- Missing favicon → letter avatar; missing screenshot → gradient placeholder;
  missing brand color → neutral accent. All fallbacks build-time, deterministic.
- External links `rel="noopener"`, no runtime fetches — fully static.

## Testing

- Vitest unit tests for `parse-readme.ts` against a committed README fixture
  (current copy) + malformed-input cases asserting loud failure.
- Contrast assertions inside `extract-colors.ts` (script fails if any derived
  token can't reach AA).
- Manual visual pass via agent-browser after first build (both themes).

## Out of scope (v1)

- Model-page screenshots, uptime/status checks, historical limit tracking,
  user accounts/favorites, comparing paid tiers.
